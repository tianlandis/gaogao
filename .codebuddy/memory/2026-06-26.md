## 6. 等位换算算法（最终确认：专业分百分位映射 + 真实数据）
- **用户最终确认**：我们暂无2026综合分段表（等真实数据来了才算圆满），不能用卷积生成的假数据
- **等位换算正确算法**（使用全部真实数据）：
  1. 用户输入文化分 + 专业分 → rawComp = culture×0.5 + pro×1.25
  2. 在 `*_art_2026` 专业分段表中找到专业排名 → 百分位
  3. 百分位映射到 `*_art_2025` 专业分段表 → 等位2025专业分
  4. equivComp = culture×0.5 + equivPro×1.25
  5. 在 `*_2025` 综合分段表查等位综合分排名
- **重要**：`score_from_rank` 条件必须是 `c1 <= rank <= c2`（低累积→高累积方向），不是 `c1 >= rank >= c2`
- 已删除 `generate_2026_comprehensive.py` 和 `compressed_ranks.json` 中假 `*_2026` 综合表
- 2024年文化/综合 .xlsx 文件实为 OLE2(.xls)格式，已改扩展名修复Windows打开问题

## 8. 蒙特卡洛回测验证 (validate_monte_carlo.py)
- 用2025年三层真实数据（art_2025 / culture_2025 / _2025）检验了"专业分与文化分独立分布"假设
- 随机打乱专业分和文化分后配对算模拟综合表 → 与真实综合表对比
- **结果**：平均绝对差4.1分（声乐2.4/器乐5.5/音教4.3），KS距离15.4分
- **系统性偏差**：模拟表高分端偏低、低分端偏高（说明专业分和文化分正相关，不独立）
- **尖端误差**：前5%选手误差17-20分，这对冲名校的学生影响太大
- **结论**：蒙特卡洛方案不可用于生产。继续保持当前专业分百分位映射算法，等2026综合分段表来了切过去
- 验证脚本 `validate_monte_carlo.py` 保留备用

## 9. 算法设计关键认知：专业表分母不可靠（用户指出）
- **用户明确指出**：2026专业分段表 8300+ 人不准确，是"参加专业考试人数"而非"参加投档人数"
- **核心问题**：等位比较必须用综合分段表对综合分段表做跨年百分位映射，不能绕道专业分段表
- **正确算法**：等2026综合分段表 → 综合表百分位 → 映射到2025综合表 → 等位综合分
- **分母意义**：综合表分母 = 最终参加投档的人，维度一致百分位才有意义
