# 01 — 项目架构总览

> 触发条件：首次接触项目、需要理解整体结构、新增文件或模块时必读。

---

## 1.1 项目定位

| 维度 | 说明 |
|------|------|
| **目标用户** | 2026年山东省音乐类统考考生 |
| **核心功能** | 输入文化分+专业分，输出按"冲/稳/保"分梯度的院校志愿推荐 |
| **数据基础** | 2025年山东省教育招生考试院官方投档/录取数据 |
| **版本** | V0.1（基于2025投档数据预测2026志愿） |
| **部署方式** | 纯静态前端 + Node.js 静态文件服务器（serve.js） |
| **访问地址** | http://localhost:8080 |

## 1.2 V0.1 核心假设

1. **暂用2025年的投档线作为唯一基准**，预测2026年考生可填报的志愿梯度
2. **等位分换算**：用2026专业分位次反推2025等位专业分，再算等位综合分
3. **综合分公式假定不变**：综合分 = 文化分 × 0.5 + 专业分 × 2.5 × 0.5（满分750）
4. **投档线年际波动在 ±5-15 分内**，按三档阈值分类（保/稳/冲）

## 1.4 文件职责边界

| 文件 | 职责 | 修改权限 |
|------|------|----------|
| `index.html` | 前端全部逻辑（UI+计算+渲染+导出） | **谨慎修改，必须先读对应文档** |
| `data.json` | 院校投档数据 `{schools: [...]}` | **Python脚本生成，不可手动编辑** |
| `compressed_ranks.json` | 位次字典 | **Python脚本生成，不可手动编辑** |
| `build_real_data.py` | **主数据构建入口** | 修改后需重新运行并验证输出 |
| `serve.js` | 静态文件服务器 | 除非端口冲突，尽量不改 |

## 1.5 部署方式

- 纯静态前端：`index.html` + 两个 JSON 数据文件
- 服务器：`node serve.js`（仅提供静态文件，无后端逻辑）
- 无数据库、无API、无构建工具
