## 6.2 V0.2 数据架构扩展

### compressed_ranks.json 新增键

```
{方向}_2026              // 2026综合分一分一段（新）
{方向}_culture_2026      // 2026文化分一分一段（新）
// {方向}_art_2026 已存在 ✓
```

### data.json 扩展

- 保持现有结构不变
- `id` 中 `_2025` 改为 `_2026`
- `line` 使用 2026 年实际投档线
- 可选新增 `line_2025` 字段保留历史数据供对比

## 6.3 V0.2 功能升级计划

| 序号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 1 | 双年对比模式 | P0 | 同时展示2025/2026投档线，用户可选年份 |
| 2 | 投档线预测 | P1 | 基于2025→2026线差预估当前年份投档线 |
| 3 | 院校备注(remark) | P1 | 从招生章程提取特殊要求（限钢琴、需校考等） |
| 4 | 录取概率排序 | P2 | 按概率而非梯度排序 |
| 5 | 数据年份切换 | P0 | 前端可选择2025或2026为基准 |

---

## 6.4 架构扩展注意事项

1. **双年数据共存**：`data.json` 中 school 对象可包含 `line_2025` 和 `line_2026` 两个字段，通过年份参数选择
2. **算法不变**：等位分4步流程不因年份切换而改变
3. **筛选不变**：筛选栏的 `batch` 下拉选项需动态根据选中年份的批次列表生成
4. **向后兼容**：V0.2 必须兼容 V0.1 的数据格式（仅 `line` 无 `line_2026` 的情况）
