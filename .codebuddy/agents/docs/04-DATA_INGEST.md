### XLS 解析规则

1. 跳过标题行（含"专业代号""院校代号""投档计划""投档最低"等关键词）
2. **列映射**：
   - 第1列：`专业代号 + 专业名称` → 正则提取 `major_code`（前部）和 `major`（后部）
   - 第2列：`院校代号 + 院校名称` → 正则提取 `code` 和 `name`
   - 第3列：投档计划数（取第一个数字）
   - 最后一列：投档最低分（提取数字和小数点）

## 4.2 标签富化规则

### 方向判定（direction）

按专业名关键词匹配：

| 方向 | 关键词 |
|------|--------|
| `vocal` | 声乐、流行演唱、音乐剧、美声、民族唱法 |
| `instrumental` | 器乐、钢琴、管弦、中国乐器、键盘、电吉他 |
| `education` | 其余全部（兜底） |

### 省份判定（province）

```
优先级1: UNI_DB 字典 — 按院校代码精确匹配（最准确）
优先级2: PROVINCE_KW 字典 — 地名关键词匹配（兜底）
```

### 性质判定（nature）

```
优先级1: UNI_DB 字典 — 按院校代码精确匹配
优先级2: 默认 "公办"
优先级3: 含以下关键词 → "民办"：
         民办、独立学院、城市学院、职业技术、职业学院
```

### 去重规则

按 `{code}_{major_code}_{batch}` 为去重 key，保留 `line`（投档线）最高的那条。

## 4.3 位次数据提取（compressed_ranks.json）

### 数据来源（山东省教育招生考试院官网）

| 表 | NewsID |
|------|--------|
| 综合分一分一段表 | 6954 |
| 文化分一分一段表（双达线） | 6952 |
| 专业分一分一段表（双达线） | 6953 |

### 文件读取

1. 支持 `.xls`（xlrd引擎）和 `.xlsx`（openpyxl引擎）和 `.csv`
2. 跳过标题行（含"成绩/分数段/本段人数/累计人数/合计"等关键词）
3. 列映射：第1列 = 分数，第3列（若无则第2列）= 累计人数

### 压缩策略

| 数据类型 | 采样间隔 | 说明 |
|----------|---------|------|
| 文化分 | 每 5 分 | 数据量大，压缩可减少文件体积 |
| 专业分 | 每 2 分 | 中等密度 |
| 综合分 | 全部保留 | 精度要求最高 |

## 4.5 V0.2 新增数据时的操作

1. 将新 XLS 文件放入 `downloads/` 目录
2. 在 `build_real_data.py` 中添加新文件路径和对应的批次/层次映射
3. 运行 `python build_real_data.py` 重新生成
4. 验证 `data.json` 和 `compressed_ranks.json` 输出正确
