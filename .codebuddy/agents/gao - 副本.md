## 一、项目定位

| 维度 | 说明 |
|------|------|
| **目标用户** | 2026年山东省音乐类统考考生 |
| **核心功能** | 输入文化分+专业分，输出按"冲/稳/保"分梯度的院校志愿推荐 |
| **数据基础** | 2025年山东省教育招生考试院官方投档/录取数据（作为基准线） |
| **版本** | V0.1（基于2025投档数据预测2026志愿） |
| **部署方式** | 纯静态前端 + Node.js 静态文件服务器（serve.js） |
| **访问地址** | http://localhost:8080 |

### V0.1 核心假设
1. **暂用2025年的投档线作为唯一基准**，预测2026年考生可填报的志愿梯度
2. **2026年等位分换算**：使用2026年专业一分一段表 + 2025年专业一分一段表，将考生的2026专业分换算为2025等位专业分
3. **综合分公式假定不变**：综合分 = 文化分 × 0.5 + 专业分 × 2.5 × 0.5（满分750）
4. **投档线年际波动在 ±5-15 分内**，按三档阈值分类（保/稳/冲）

### 3.2 compressed_ranks.json 数据结构（位次字典）

```json
{
  "vocal_art_2026":       { "178": 8358, "180": 8297, ... },  // 2026声乐专业一分一段
  "vocal_art_2025":       { "190": 5000, "188": 5200, ... },  // 2025声乐专业一分一段
  "vocal_2025":           { "620.50": 1, "615.00": 5, ... },  // 2025声乐综合分一分一段
  "vocal_culture_2025":   { "604": 1, "589": 2, ... },        // 2025声乐文化分一分一段
  "instrumental_art_2026": { ... },                            // 同上，器乐方向
  "instrumental_art_2025": { ... },
  "instrumental_2025":     { ... },
  "instrumental_culture_2025": { ... },
  "education_art_2026":   { ... },                             // 同上，音教方向
```

**命名规则（严格）**：`{方向}_{数据类}_{年份}`
- 方向：`vocal` / `instrumental` / `education`
- 数据类：`art`（专业分一分一段）| `culture`（文化分一分一段）| 无后缀（综合分一分一段）
- 年份：`2025` / `2026`
- 键：分数（string），值：累计人数（int），按分数降序排列

## 四、核心算法

### 4.1 等位分标准化（4步流程）⭐ 核心

```
用户输入：文化分(culture) + 专业分(pro) + 方向(dir)

STEP 1 — 原始综合分
  rawComp = culture × 0.5 + pro × 2.5 × 0.5

STEP 2 — 2026专业位次
  art2026 = rankData[dir + '_art_2026']     // 查2026专业一分一段表
  artRank = 插值查表(art2026, pro)          // 得到该专业分对应的全省位次

STEP 3 — 等位专业分换算（位次回推2025分数）
  art2025 = rankData[dir + '_art_2025']     // 查2025专业一分一段表
  equivPro = 位次反查(art2025, artRank)     // 用2026的位次反推2025等位分

STEP 4 — 等位综合分 & 综合位次
  equivComp = culture × 0.5 + equivPro × 2.5 × 0.5
  comp2025 = rankData[dir + '_2025']        // 查2025综合分一分一段表
  compRank = 插值查表(comp2025, equivComp)  // 得到综合位次
```

**为什么需要等位分？**
因为每年考题难度不同，直接比原始分没有意义。通过"位次"这个不变量，将2026年的专业分映射到2025年的分数刻度上，这样才能与2025年的投档线公平比较。

### 4.2 插值算法

```javascript
// 分数→位次 插值查询
function rankFromDict(dict, score):
    将dict按分数降序排序
    二分查找score所处区间
    线性插值计算位次

// 位次→分数 反向查询
function scoreFromRank(dict, rank):
    将dict按分数降序排序
    二分查找rank所处区间
    线性插值计算分数
```

### 4.3 梯度计算（冲/稳/保）

```javascript
diff = 等位综合分 - 学校投档线(line)

if diff > 15:    → "保" (tag-bao, 绿色)   // 超过投档线15分以上
if diff >= 0:    → "稳" (tag-wen, 蓝色)   // 达线但不超过15分
if diff >= -5:   → "冲" (tag-chong, 橙色) // 低于线但差距在5分以内
otherwise:       → 不显示                    // 差距超过5分，不建议
```

### 4.4 文化课超越率

```javascript
cultureRank = 插值查表(cultureDict, culture分)
cultureTotal = max(所有累计人数)
culturePct = (1 - cultureRank / cultureTotal) × 100
```

含义：在双达线考生中，你的文化分超过了百分之多少的人。

## 五、数据提取与清洗规则

### 5.1 院校投档数据提取（build_real_data.py）

**输入**：6个官方XLS文件（downloads/目录下）

| 文件 | NewsID | 批次 | 层次 |
|------|--------|------|------|
| 山东省2025年艺术类本科批音乐类第1次志愿投档情况表(公布).xls | 6986 | 本科批第1次投档 | 本科 |
| 山东省2025年艺术类本科批音乐类第1次志愿投档情况表(公布)（补充）.xls | 6986 | 本科批第1次投档 | 本科 |
| 山东省2025年艺术类本科批音乐类第1次志愿录取情况表.xls | 6989 | 本科批第1次录取 | 本科 |
| 山东省2025年艺术类本科批音乐类第2次志愿投档情况表.xls | 7001 | 本科批第2次投档 | 本科 |
| 山东省2025年艺术类专科批音乐类第1次志愿投档情况表.xls | 7009 | 专科批第1次投档 | 专科 |
| 山东省2025年艺术类专科批音乐类第2次志愿投档情况表.xls | 7018 | 专科批第2次投档 | 专科 |

**解析规则**：
1. 跳过标题行（含"专业代号""院校代号""投档计划""投档最低"等关键词）
2. 第1列：`专业代号 + 专业名称`，正则提取 `major_code` 和 `major`
3. 第2列：`院校代号 + 院校名称`，正则提取 `code` 和 `name`
4. 第3列：投档计划数（取第一个数字）
5. 最后一列：投档最低分（提取数字和小数点）

**标签富化规则**：
- **方向判定**：专业名包含"声乐/流行演唱/音乐剧/美声/民族唱法"→vocal；包含"器乐/钢琴/管弦/中国乐器/键盘/电吉他"→instrumental；其余→education
- **省份判定**：优先查 `UNI_DB` 字典（院校代码精确匹配），未命中则用 `PROVINCE_KW` 关键词匹配（地名→省份映射）
- **性质判定**：优先查字典，否则默认"公办"，含"民办/独立学院/城市学院/职业技术/职业学院"→"民办"
- **去重**：按 `code_majorCode_batch` 为key去重，保留投档线最高的记录

### 5.2 位次数据提取（compressed_ranks.json）

**数据来源（山东省教育招生考试院官网）**：
- 综合分一分一段表：NewsID=6954
- 文化分一分一段表（双达线）：NewsID=6952
- 专业分一分一段表（双达线）：NewsID=6953

**读取方式**：
1. 支持 `.xls`（xlrd引擎）和 `.xlsx`（openpyxl引擎）和 `.csv`
2. 跳过标题行（含"成绩/分数段/本段人数/累计人数/合计"等关键词）
3. 第1列 = 分数，第3列（或第2列）= 累计人数

**压缩策略**：
- 文化分：每5分采样（interval=5）
- 专业分：每2分采样（interval=2）
- 综合分：保留全部（不压缩）

## 六、前端交互规范

### 6.1 状态机

```
初始状态 → 点击"开始智能预测" → 计算展示状态 → 筛选/导出状态
                ↑                                              |
                └──────── 再次点击"开始智能预测" ←──────────────-┘
```

### 6.2 筛选防御规则

- `applyFilters()` 必须检查 `state.calcResult` 是否存在，不存在则直接 return（不渲染）
- 点击"开始智能预测"时必须重置所有筛选条件为默认值
- 所有 DOM 操作前必须检查元素是否为 null

### 6.3 渲染防御规则

- `renderResults / renderSchoolList / updateStatsBar` 所有函数必须 try-catch 包裹
- 所有 `E.xxx` 属性访问前加 `if(E.xxx)` null 检查
- 学校列表按"批次"分组，批次排序：本科批第1次投档 > 本科批第1次录取 > 本科批第2次投档 > 专科批第1次投档 > 专科批第2次投档

### 6.4 响应式规则

- 竖屏（portrait）或宽度≤768px：输入面板在上、结果在下，学校卡片横向滑动
- 横屏（landscape）且高度≤768px：左右分栏，输入面板 sticky 定位，结果面板可滚动
- 默认布局：左右分栏（380px 输入 + flex 结果）

## 八、修改代码的强制规则

1. **禁止删除或修改 `compressed_ranks.json` 和 `data.json` 的顶级键名结构**
